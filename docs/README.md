## 🎯 Принципы Документации

### Что Мы Документируем

✅ **Архитектурные решения** - почему выбрали такую структуру
✅ **SOLID примеры** - как правильно писать код
✅ **Частые задачи** - пошаговые инструкции
✅ **Философия проекта** - принципы и подходы

❌ **НЕ документируем:**
- Очевидные вещи из кода
- API конкретных методов (смотри JSDoc в коде)
- Детали implementation (читай код напрямую)

### Стиль Документации

- 📝 **Кратко и по делу** - без воды
- 🎯 **Практические примеры** - больше кода, меньше текста
- ✅ **Правильно vs ❌ Неправильно** - учим на контрасте
- 🔍 **Quick navigation** - легко найти нужное

## 🔄 Обновление Документации

### Когда Обновлять?

Обновляй документацию при:
- Добавлении новой папки в структуру
- Изменении архитектурного подхода
- Рефакторинге больших систем
- Добавлении нового паттерна

### Как Обновлять?

1. Отредактируй соответствующий `.md` файл
2. Обнови "Последнее обновление" в конце файла
3. Добавь версию если это breaking change

## ❓ FAQ

**Q: Нужно ли читать всю документацию?**
A: Начни с [`overview.md`](./architecture/overview.md). Остальное - по необходимости.

**Q: Документация устарела?**
A: Проверь "Последнее обновление" в конце каждого файла. Если сомневаешься - спроси команду.

**Q: Могу ли я добавить свою документацию?**
A: Да! Создай новый `.md` файл и добавь ссылку в этот README.

**Q: Документация на русском?**
A: Да, вся документация на русском для удобства команды.
